jRender Documentation

Simple Example

To generate your form, call jRender with your JSON Schema and jRender options

Code

var personal_details_schema = {
	type : "object",
	title : "Personal Details",
	properties : {
		"First Name" : {
			"type" : "string"
		},
		"Last Name" : {
			"type" : "string"
		},
		"Address" : {

			"properties" : {
				"Door No." : {
					"type" : "string"
				},
				"Street Name" : {
					"type" : "string"
				},
				"City" : {
					"type" : "string"
				},
				"Country" : {
					"type" : "string"
				},
				"PIN" : {
					"type" : "string"
				}
			}
		},
		"Age" : {
			"type" : "string"
		},
		"Interests" : {
			"enum" : ["Sports", "Music", "Movies", "Food", "Fashion", "Technology", "Business"]
		},
		"Gender" : {
			"type" : "boolean",
			"format" : "Male/Female"
		}
	}
};

var json_to_render = personal_details_schema;

options = {
    "method" : "",
    "action" : "",
    "root" : "Personal Details",
    "hook" : "#simple_example_output",
    "render-options" : {
        "render-types" : {
            "enum" : "checkbox",
            "boolean" : "radio"
        }
    }
};

jrender = jRender(json_to_render, options);

Setting Options

You can pass the following options to jRender

method
The type of form submit : POST / GET
action
Where the form should submit to
root
The title of the form to be rendered
hook
id / class of the DOM element where the form has to be rendered
render-options : render-types
You can map types like "enum" and "boolean" to specific DOM types like "select", "checkbox" or "radio"

Advanced Example

Use $ref to reference definitions in your JSON Schema. Arrays in jRender are handled by rendering buttons which can be clicked to render the actual form sections.

Code

var personal_details_schema = {
	type : "object",
	title : "Personal Details",
	properties : {
		"First Name" : {
			"type" : "string"
		},
		"Last Name" : {
			"type" : "string"
		},
		"Address" : {
			"type" : "array",
			"items" : {
				"$ref" : "#/definitions/address_object"
			}
		},
		"Age" : {
			"type" : "string"
		},
		"Interested In" : {
			"enum" : ["Sports", "Music", "Movies", "Food", "Fashion", "Technology", "Business"]
		},
		"Gender" : {
			"type" : "boolean",
			"format" : "Male/Female"
		}
	},
	definitions : {
		"address_object" : {
			"properties" : {
				"Door No." : {
					"type" : "string"
				},
				"Street Name" : {
					"type" : "string"
				},
				"City" : {
					"type" : "string"
				},
				"Country" : {
					"type" : "string"
				},
				"PIN" : {
					"type" : "string"
				}
			}
		}
	}
};

var json_to_render = personal_details_schema;

options = {
    "method" : "",
    "action" : "",
    "root" : "Personal Details",
    "hook" : "#advanced_example_output",
    "render-options" : {
        "render-types" : {
            "enum" : "select",
            "boolean" : "radio"
        }
    }
};

jrender = jRender(json_to_render, options);

Self Referencing Example

Self referencing enables properties of objects to refer to self / some parent in the tree. Self referecing in jRender is handled by enabling lazy loading through button clicks.

Code

	type : "object",

	properties : {
		"First Name" : {
			"type" : "string"
		},

			"type" : "array",
			"items" : {

		},
		"Child" : {
			"$ref" : "#"

	definitions : {
